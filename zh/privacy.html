2. 产品与开源参考说明（TeslaMate）

TeLog 在数据模型、本地记录逻辑、统计面板概念与地理编码回退策略等方面， 参考了开源项目 TeslaMate（teslamate-org/teslamate）的公开设计与实现思路。 TeslaMate 是面向特斯拉车主的自托管车辆记录方案，其源代码在开源许可证下公开。

  • TeLog 为独立开发的移动应用，并非 TeslaMate 官方客户端或其二进制分发版本，也不要求用户自行部署 TeslaMate 服务器。
  • TeLog 针对中国大陆场景进行了独立实现：中国区 Fleet API（auth.tesla.cn）、高德地图、本地 SQLite 加密、可选境内云端代采、App Store 订阅等，上述能力不属于 TeslaMate 上游项目的默认可用范围。
  • 对开源项目中受许可证保护的部分，我们仅在许可证允许范围内参考或使用；TeLog 的界面、增值报告（电池保值、驾驶行为等）、订阅与云端架构为独立设计与运营。
  • TeslaMate 商标与项目名归其权利人所有；本政策提及 TeslaMate 仅为说明技术参考来源，不构成隶属或背书关系。

与您的隐私相关：参考开源架构并不意味着您的数据会流向 TeslaMate 项目或任何自托管社区服务器； 默认情况下，您的车辆数据仅保存在您自己的手机（见第 8 节）。

3. 中国合规开发原则

TeLog 在中国大陆提供服务的核心合规设计如下：

  • 合法正当必要：仅处理实现车辆档案、行程充电记录、电池分析等功能所必需的信息，不过度收集。
  • 默认本地、单独同意：车辆遥测数据默认存于本机；上传云端、家庭位置共享等需另行明示同意。
  • 境内存储（可选云端）：经您同意后上传的云端副本存储在中国境内服务器（见第 9 节）。
  • 官方 API 渠道：车辆数据通过特斯拉中国区官方开发者平台取得的 Fleet API 能力读取，拒绝非官方爬取接口。
  • 正规地图服务：在中国境内使用高德开放平台依法申请的 Key 提供地图展示与逆地理编码（见第 5 节）。
  • 不做广告跟踪：不集成广告 SDK，不出售个人信息，不在 App 内跨应用跟踪您。
  • 用户权利保障：提供导出、删除、撤回同意与投诉渠道（见第 12 节）。

4. 特斯拉中国区 Fleet API 数据链路

TeLog 通过运营者在 developer.tesla.cn 正规注册的第三方应用，在您授权后读取车辆数据。完整链路如下：

环节说明是否经 TeLog 业务服务器中转车辆遥测
开发者注册 运营者在特斯拉中国开发者平台注册应用、遵守其协议并取得 client_id 与 Fleet API 资格 —
车主授权 您跳转 auth.tesla.cn 官方页面登录并勾选 OAuth 权限范围 否（特斯拉官方认证）
Token 换票 OAuth 授权码 / refresh token 经 familymedvault.com/api/telog/token 代理换取 access token；client_secret 仅存服务端 仅令牌交换，不含车辆遥测内容
车辆数据拉取 App 直连 fleet-api.prd.cn.vn.cloud.tesla.cn 调用 vehicle_data 等接口 否（与 TeslaMate 自托管直连特斯拉的方式一致）
本地处理 数据解析、行程/充电记录生成、统计与展示默认在您的 iPhone 本机完成 否
可选云端代采 仅在您单独同意后，境内服务按授权代拉 Fleet API 并加密存储副本 是（可选功能，见第 8 节）

4.1 我们可能申请的 OAuth 范围

以特斯拉授权页实际展示为准，通常包括：

  • openid、offline_access、user_data
  • vehicle_device_data — 车辆状态、电池、充电、里程、软件版本等
  • vehicle_location — 车辆位置（行程轨迹、地理围栏）
  • vehicle_cmds、vehicle_charging_cmds — 仅在您主动使用唤醒、预空调、定时充电等指令功能时

我们不会收集或存储您的特斯拉账号密码。 您可随时在特斯拉账户或本 App 中断开授权。

4.2 自 Fleet API 产生的信息类型

  • 车辆识别：VIN、显示名称、在线/休眠/行驶/充电状态
  • 位置与行驶：GPS 坐标、航向、速度、档位、功率；衍生的行程与轨迹点
  • 电池与续航：SOC、续航估算、里程表
  • 充电：充电状态、功率/电流/电压、已充入电量、充电上限、预计充满时间
  • 车辆状态：OTA、车门/尾箱、锁车、哨兵、胎压、车内是否有人
  • 空调：车内外温度、预空调等
  • 您在 App 内创建的内容：地理围栏名称、地址标注、档案备注、分享设置等

上述信息中，精确位置、行程轨迹、VIN 等可能构成《个人信息保护法》下的敏感个人信息；我们仅在具有明确、合理目的且取得您的单独同意（连接车辆、开启相关功能）后处理。

5. 高德地图服务合规使用

为在中华人民共和国境内合法提供地图展示、围栏编辑与地址文字化（逆地理编码），TeLog 使用 高德开放平台（lbs.amap.com） 提供的服务，并遵守《中华人民共和国测绘法》及地图管理相关规定。

5.1 Key 申请与使用方式

  • 运营者通过高德开放平台实名注册开发者账号，为 TeLog 依法申请并使用 API Key，绑定对应服务平台类型，不超范围调用。
  • Web 端 JS API（地图展示）：用于围栏编辑、行程/到访地图 WebView 展示；按高德要求配置 Key 与安全密钥（security code）。
  • Web 服务 API（逆地理编码）：用于将车辆 GPS 坐标（WGS-84）转换为国内地图坐标系（GCJ-02）后，向高德请求地址文字描述。
  • 当前实现不使用高德 iOS 原生 SDK，而采用 App 内 WebView + REST 方式，与 Key 申请类型一致。

5.2 向高德传输的数据

为获取地址或渲染地图，App 可能向高德传输：

  • 经纬度坐标（经国标转换后的 GCJ-02）
  • 地图缩放、视野范围等渲染所需参数
  • 高德为提供服务所需的设备/网络技术信息（以其隐私政策为准）

我们不会向高德传输您的特斯拉账号密码、OAuth token 或完整行程数据库。

5.3 境外回退（OpenStreetMap Nominatim）

当高德服务不可用且您在设置中允许时，App 可能将坐标回退请求至 OpenStreetMap Nominatim（境外服务）进行逆地理编码。 该回退非默认首选；若触发，将向境外服务传输经纬度坐标。 您可在高级设置中选择地理编码提供方偏好。

高德隐私政策请参阅：高德开放平台隐私权政策。

8. 本地优先与可选云端

8.1 默认：仅本机

首次使用时，我们将告知「隐私与本地优先」原则。在您连接特斯拉后，Fleet API 返回的数据 默认仅保存在您 iPhone 的加密 SQLite 数据库中，不上传至我们的业务服务器进行存储或分析。

8.2 可选：云端代采（TeLog Plus）

仅在您阅读《云端代采数据处理说明》并单独勾选同意，且通常持有有效 TeLog Plus 资格时：

  • 境内服务可能代您调用 Fleet API，补采手机离线期间的行程与充电数据；
  • 上传类别包括：行程、轨迹点、充电过程、状态事件、车辆快照、围栏与地址标注等（与 TeslaMate 核心表项对应）；
  • 用途：连续记录、统计、备份与跨设备同步；不用于广告；
  • 您可在「设置 → 隐私与数据」或高级设置中随时关闭；关闭后停止上传并删除境内副本（本地数据保留）。

8.3 可选：家庭位置共享

与云端代采分开同意。仅当您确认《家庭位置共享说明》后，受邀且获「位置」权限的成员方可查看车辆实时位置；您可随时撤销邀请。

9. 存储地点与期限

  • 本机数据：保存在您的 iPhone，期限至您卸载 App、清空本地数据或更换设备并删除备份为止。
  • OAuth token：加密保存在本机；refresh 交换经 HTTPS 代理完成。
  • 云端副本（如已开启）：存储在中华人民共和国境内；默认保留 365 天滚动周期，关闭云端代采后删除服务器副本。
  • 支持邮件：为处理工单合理保留，一般不超过 24 个月，除非法律要求更长保留。
  • 法律合规留存：在争议、诉讼或监管要求下，可能在必要范围内延长保留。

13. 未成年人保护

  • 本机 SQLite 加密存储与系统级设备保护（Face ID / 密码由您设备负责）；
  • OAuth client_secret 不进 App 安装包；HTTPS 传输；
  • 云端副本加密存储与访问控制（启用云端时）；
  • 最小权限原则：Fleet API 仅请求功能所需 scope；
  • 安全事件：将按法律规定告知并采取补救措施。

互联网环境并非绝对安全，请妥善保管设备与 Apple ID。

15. 跨境与境外回退
